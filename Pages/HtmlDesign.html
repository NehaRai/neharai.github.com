Best Practices for HTML design

Today I gone through the best practices for designing of a web page and I explored lots of facts, which can consider before start designing a page. Best practices in designing of web pages is all about writing optimized code that is valid HTML, which make it easier to correct problems, and edit pages.
 HTML is the structure of a page. HTML stands for "Hyper Text Markup Language”.
  • Hyper is the opposite of linear. Old-fashioned computer programs were necessarily linear - that is, they had a specific order. But with a "Hyper" language such as HTML, the user can go anywhere on the web page at any time.
  • Text is just what you're looking at now - English characters used to make up ordinary words.
  • Markup is what is done to the text to change its appearance. For instance, "marking up" your text with <b> before it and </b> after it will put that text in bold.
  • Language HTML is the language that computers read in order to understand web pages.
Following are few best practices, which guide us and allow us to approach clean and correct markup –
Declare a Doctype
The doctype declaration should be the first thing in HTML documents. It refers to Document Type Definition (DTD). The DTD specify the rule for markup.
Always close the tags
It is best practice to always close the tag. Each time the browser finds a tag, it is displayed accordingly (paragraphs look like paragraphs, tables look like tables, etc). Tags have 3 major parts: opening tag(s), content(s), and closing tag(s). A completed tag is termed an element.
Declare meaningful Title
The <title> tag helps make a web page more meaningful. It contains the name of the web page. The most important thing to understand about the title tag is that it is not part of the web page. The title tag is for other applications, such as:
  • It appears in the browser title bar - at the very top of the window. If you look at browser title bar we should see the name of our browser and the title of this page - "HTML Title Tag".
  • If a visitor adds your page to their favorites (bookmarks), the page title is the default name it will give the bookmark.

Use Divs to Divide Your Layout into Major Sections

The <div> tag defines logical divisions (defined) of Web page. We could define a section of our page as a call out and give that section a different style from the surrounding text. It will also help us avoid confusion and excess use of divs, especially if we are writing complex and lengthy markup.

Separate Content from Presentation

HTML is content and CSS provides content’s visual presentation. Always use both separately. Always create a separate CSS file for styles. It will help us and future developers that might be work on same code make changes to the design quicker and make content more digestible for user agents.

Use Heading Elements Wisely

Learn to use <h1> to <h6> elements to denote your HTML’s content hierarchy. This helps make content more meaningful for screen-reading software and search engines, as well as other user agents. 

Close Your Tags

Always close the tags. Some browsers may still render your pages correctly (under Quirks mode), but not closing your tags is invalid under standards.

Use Lower Case Markup

Lowercase markup compresses more efficiently than uppercase markup. It is an industry-standard practice to keep markup lower-cased. Capitalizing markup will work and will probably not affect how web pages are rendered, but it does affect code readability.

Use Alt Attributes with Images

The required alt attribute specifies an alternate text for an image, if the image cannot be displayed. The alt attribute provides alternative information for an image if a user for some reason cannot view it (because of slow connection, an error in the src attribute, or if the user uses a screen reader).

Use an Unordered List (<ul>) for Navigation

An unordered list (the <ul/> tag in HTML) is a great way to easily organize your site navigation with a minimal use of HTML. By far the easiest, cleanest, most lightweight and most flexible way to create a horizontal navigation menu is to use an unordered list and style it with CSS.

Use Fieldset and Labels in Web Forms

Use the <label> element to label input fields. Divide input fields into logical sets using <fieldset>. Name a <fieldset> using <legend>. All of this will make forms more understandable for users and improve the quality of code.

Do proper commenting on code

Probably the most important piece of code you can write is a comment. Comments not only make code more clearly to others, it will also help to remember what you were thinking for future development.
