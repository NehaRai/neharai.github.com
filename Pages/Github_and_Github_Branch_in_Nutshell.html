Github and Github Branch in Nutshell

When you ask me what is Github, I would say Github is a "Wikipedia for programmers", where programmer  can create and edit files,share code, see who changed what, view old versions of files/code, and access it from anywhere in the world. Your code is just a click away.

Today I am going to speak about "Github Branches".
Branches are like; we divide our main line into sub lines and continue to do work without messing with the main line.
   
Following are some of the commands, which used to perform branching operations-

Create branch

 To view our available branches, we run the ‘git branch’ command with  no arguments.

                   Input:  $ git branch
                   Output: * master
 
             We can see that we only have one branch ‘master’ and the ’*’ 
             indicates that we are currently on it.
 
To create a new branch, we can use ‘git branch (branchname)’, which  will create a branch at the point we’re currently at.

                   Input:  $ git branch TestBranch
                   Output:  It creates new branch TestBranch
                                       
                                       
To switch to that branch so that the work we do is saved to it instead of the ‘master’ branch, we run the ‘git checkout’ command.

                   Input:    $ git checkout TestBranch
                   Output:  Switched to branch "TestBranch "
                   
                   Input:    $ git branch
                   Output:  * TestBranch
                                 master
 
           
Working with branch
 
Add a ‘newfile’ file, make a change to the ‘oldfile.txt’ file, and then make a commit with both changes in it.

             $ vi old/oldfile.txt
             $ vi newfile
             $ git add newfile
             $ git commit -m 'added newfile’

 Now if we take a look, we can see that we have 2 files and one subdirectory.
 
              Input:   $ ls
              Output: README
                           Newfile
                           old
 
    So, let’s now suppose that we need to go back and make a fix on our original version of the oldfile.txt file. We     can revert back to where our project was before we branched with the ‘checkout' command
 
          Input:    $ git checkout master
          Output:  Switched to branch "master"
         
          Input :   $ ls
          Output:  README
                        old

  Now we can see that our working directory no longer has the newfile file it in that’s because the master branch
      didn’t have that file. If we do a commit here and then switch back, we’ll see the newfile file there again, and the       oldfile.txt file reverted back to where we left it in the TestBranch branch.
 
                $ vi old/oldfile.txt                                                        
                $ git commit -m 'added a commit function'
                $ git checkout TestBranch
                      Switched to branch " TestBranch "
                Input:    $ ls
                Output:  README
                              newfile          
                              old
 
 Merging Branches
 
To merge branches run the ‘git merge’ command.
There are two cases-
             Merging will be done cleanly-
               
              Input: $ git checkout master
                        $ git merge TextBranch
                        $ git push
              Output: Auto-merging old/oldfile.txt
                         Merge made by recursive.
                        Old/oldfile.txt |    1 + 1 files changed, 1 insertions(+),
                        0 deletions(-)
 
             Get a merge conflict

                
                Input:   $ git checkout master            
                    $ git merge TextBranch
                    $ git push              
      Output:  Auto-merging old/oldfile.txt
                   CONFLICT (content): Merge conflict in ld/oldfile.txt
              Automatic merge failed; fix conflicts and then commit.

If I get a merge conflict. In this case when merge is not happen cleanly, resolve the conflicts manually. Just open up the file and do it manually. After fixing we have to just add a file to re-stage it, which marks it as solved. 
             
        Input:  $ git add old/oldfile.txt
                           $ git commit
                           $ git merge TestBranch
                           $ git push
                Output:Auto-merging old/oldfile.txt
                         Merge made by recursive.
